# Device Activity Monitor — Indigo Plugin

**Version**: 1.9.0
**Author**: CliveS
**Platform**: Indigo 2025.2 / macOS / Python 3.13
**Plugin ID**: `com.clives.indigoplugin.deviceactivitymonitor`

> v1.9.0 renamed this plugin from **Sensor Monitor** → **Device Activity Monitor**.
> Same lineage; the new name better describes both halves of what it now does:
> per-device change **logging** and group-change custom **triggers**.

---

## Overview

Device Activity Monitor watches Indigo devices and variables and reacts to their state
changes two ways:

1. **Logs** changes to the Indigo event log with millisecond-precision timestamps and
   custom labels (`Front Door OPEN`, `Lux Level: 450 -> 520`, etc.).
2. **Fires custom triggers** when any device in a named **Group** changes state, with
   an optional direction filter (any / activated / deactivated). Groups are
   first-class Indigo devices managed via a rich two-list ConfigUI — no more
   editing JSON to add or remove members.

Both halves run from a single `indigo.devices.subscribeToChanges()` subscription, so
adding a device to a group or to the logger costs nothing in Indigo trigger overhead.

---

## Features

**Logging**
- Single config file lists every device + state to log
- `#` comment lines to disable entries without deleting them
- Multiple states per device (PIR + mmWave on the same sensor → two log lines)
- Custom on/off labels (OPEN/CLOSED for contacts, DETECTED/CLEAR for motion)
- Millisecond timestamps
- Variable value-change logging (`old -> new` format)
- Startup validation — warns about missing device IDs in config
- Rename and deletion detection

**Groups + Triggers**
- "Device Activity Monitor Group" — first-class Indigo device type
- Rich Add/Remove ConfigUI with folder filter on the available-devices list
- "Device Activity Monitor: Group Changed" custom trigger fires on any state change
  in the selected group
- **Direction filter**: fire on any change / only when a member becomes
  ON / OPEN / detected / only when a member becomes OFF / CLOSED / clear
- Optional save-firing-device-to-variable for use in trigger actions
- Group device exposes `memberCount`, `lastFiringDevice`, `lastFiringTime`,
  `lastFiringDirection`, and a display status state for control pages

**Discovery**
- Plugin menu **Discover All Devices** scans every Indigo device, classifies
  contact / motion candidates, and writes a ready-to-use config file
- Zigbee2MQTT-Bridge aware: reads authoritative `has_contact` / `has_occupancy`
  / `has_presence` / `has_pir` ownerProps from Z2M devices for correct classification

## Installation

1. Download `Device_Activity_Monitor.indigoPlugin.zip` from the [latest release](https://github.com/Highsteads/DeviceActivityMonitor/releases)
2. Unzip → double-click `Device_Activity_Monitor.indigoPlugin` — Indigo prompts to install
3. Plugins → Manage Plugins → enable **Device Activity Monitor**
4. Check the event log for the startup banner

---

## Configuration file location

```
<install>/Preferences/Plugins/com.clives.indigoplugin.deviceactivitymonitor/
├── device_activity_monitor_config.json   ← edit this
└── device_discovery.json                 ← generated by Discover Devices
```

`<install>` resolves via `indigo.server.getInstallFolderPath()` so the path tracks
your active Indigo version automatically.

---

## Creating a Group (for triggers)

1. **Devices → New Device**
2. **Type** = "Device Activity Monitor → Device Activity Monitor Group"
3. Pick a folder filter to narrow the Available list
4. Multi-select devices in the Available list, click **Add to Group ↓**
5. Save

To edit later, double-click the Group device — same dialog. Add or Remove without
touching JSON, with all the folder filtering and search that Indigo's native
device pickers provide.

---

## Wiring a trigger to a Group

1. **New Trigger** → Type = "Device Activity Monitor: Group Changed"
2. **Group**: pick from the dropdown (Indigo's native device picker, filtered
   to Group devices, with live member counts)
3. **Fire on**:
   - **Any change** — fires on every state update of any member (default)
   - **Any device becomes ON / OPEN / detected** — only on onState `False → True`
   - **Any device becomes OFF / CLOSED / clear** — only on onState `True → False`
4. (Optional) **Save firing device** → variable, so trigger actions can read
   which member triggered

Single-device triggers: make a Group with one device. The direction filter still
works.

## Logging config

`device_activity_monitor_config.json` has two arrays:

```json
{
  "devices": [
    {"id": 123456789, "name": "Front Door",  "state": "onState", "label": "Front Door",  "on_text": "OPEN", "off_text": "CLOSED"},
    {"id": 987654321, "name": "Basin mmWave","state": "pirDetection", "label": "PIR"},
    {"id": 987654321, "name": "Basin mmWave","state": "presence",     "label": "mmWave Presence"}
  ],
  "variables": [
    {"id": 241032502, "name": "Lux_Level", "label": "Lux Level"}
  ]
}
```

- `#` at the start of a line disables that entry
- Trailing commas before `]` or `}` are auto-stripped
- Multiple rows with the same `id` monitor multiple states on the same device
- Apply changes with **Plugins → Device Activity Monitor → Reload Config File**

### Field reference

| Field      | Required | Description                                                        |
|------------|----------|--------------------------------------------------------------------|
| `id`       | Yes      | Indigo device or variable ID (integer)                             |
| `name`     | No       | For your reference only                                            |
| `state`    | Yes      | `"onState"` reads `device.onState`; otherwise reads from `device.states` |
| `label`    | No       | Log text after the device name (defaults to `name`)                |
| `on_text`  | No       | Log text when state is True (default `ON`)                         |
| `off_text` | No       | Log text when state is False (default `OFF`)                       |

## Migration notes for upgraders from "Sensor Monitor"

If you ever installed a pre-v1.9.0 release under the old name:

1. Disable **Sensor Monitor** in Indigo Manage Plugins
2. Delete the old plugin bundle from `<install>/Plugins/Sensor_Monitor.indigoPlugin/`
3. Install **Device Activity Monitor** v1.9.0
4. Move `<install>/Preferences/Plugins/com.clives.indigoplugin.sensormonitor/` →
   `<install>/Preferences/Plugins/com.clives.indigoplugin.deviceactivitymonitor/`
5. Rename `sensor_monitor_config.json` → `device_activity_monitor_config.json` inside
   that folder
6. Existing smGroup devices are not auto-migrated to damGroup; re-create groups
   via Devices → New Device (or stay on the v1.8.x branch indefinitely if preferred)
